Syarat Algoritma

  1. Ada input dan output
  2. Pasti (Definiteness)
  3. Jelas dan tidak ambigu
  4. Akhir (Finiteness)
  5. Efektif

Mencari Bilangan Terbesar dan Terkecil

Algoritma untuk mencari bilangan terbesar dan terkecil dari beberapa bilangan:

1. Masukkan tiga bilangan: A, B, dan C
2. Jika A > B dan A > C, maka A adalah terbesar
3. Jika B > A dan B > C, maka B adalah terbesar
4. Jika C > A dan C > B, maka C adalah terbesar
5. Lakukan hal serupa untuk mencari yang terkecil

Pseudocode, Input & Output

Pseudocode adalah penulisan algoritma menggunakan bahasa sehari-hari yang mudah dipahami.

Input: Data yang dimasukkan ke program
Output: Hasil keluaran dari program

Contoh Pseudocode:
Mulai
Input A, B, C
Jika A > B dan A > C, maka Tulis "A terbesar"
Jika B > A dan B > C, maka Tulis "B terbesar"
Jika C > A dan C > B, maka Tulis "C terbesar"
Selesai

Bentuk Penyajian Algoritma

  • Deskriptif (narasi langkah-langkah)
  • Pseudocode (kode semu)
  • Flowchart (bagan alir)
  • Program (kode nyata)

Simbol Flowchart

Simbol Nama Fungsi
Terminator Mulai/Selesai
Proses Langkah Proses
Input/Output Data masuk/keluar
Decision Percabangan/Keputusan
Arrow Arah alur

Mencari Rata-rata Bilangan dan Simbol Flowchart

Langkah mencari rata-rata:

  1. Masukkan n bilangan
  2. Jumlahkan semua bilangan
  3. Bagi jumlah dengan n
  4. Keluarkan hasil rata-rata

Karakteristik Algoritma

  • Jelas dan tidak ambigu
  • Memiliki input dan output
  • Efektif dan efisien
  • Berhingga (ada akhirnya)
  • Tersusun logis

Pengenalan Pola

Pengenalan pola adalah kemampuan mengenali urutan, bentuk, atau struktur tertentu dalam data atau masalah.

Contoh: Deret angka, pola segitiga, dll.
